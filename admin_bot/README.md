# Admin Bot — Интерфейс управления LeadHunter

## Описание

Admin Bot — это Telegram-бот для управления системой LeadHunter. Предоставляет удобный интерфейс для:
- Управления чатами для мониторинга
- Управления Telegram-аккаунтами для отправки сообщений
- Просмотра и обработки лидов
- Получения статистики

## Архитектура

```
admin_bot/
├── main.py              # Точка входа, инициализация бота
├── filters.py           # Пользовательские фильтры (OperatorFilter)
├── keyboards.py         # Генераторы клавиатур
├── states.py            # FSM состояния
├── lead_card.py         # Формирование лид-карточек
└── handlers/            # Обработчики команд и callback
    ├── __init__.py      # Настройка роутеров
    ├── start.py         # Команды /start, /help, /menu
    ├── chats.py         # Управление чатами
    └── accounts.py      # Управление аккаунтами
```

## Основные компоненты

### Фильтры (filters.py)

**OperatorFilter** — пропускает только сообщения от авторизованного оператора (по `OPERATOR_USER_ID`).

### Клавиатуры (keyboards.py)

Генераторы inline-клавиатур для различных меню:
- `get_main_menu_keyboard()` — главное меню
- `get_chats_menu_keyboard()` — меню чатов
- `get_accounts_menu_keyboard()` — меню аккаунтов
- `get_lead_card_keyboard()` — клавиатура лид-карточки
- `get_pagination_keyboard()` — пагинация списков

### Состояния FSM (states.py)

- `AddChatStates` — состояния добавления чата
- `AddAccountStates` — состояния добавления аккаунта
- `LeadStates` — состояния работы с лидом

### Лид-карточки (lead_card.py)

Функции для формирования красивых карточек лидов:
- `format_lead_card()` — генерация текста карточки
- `get_lead_card_keyboard()` — клавиатура действий
- `format_leads_list()` — список лидов

### Обработчики (handlers/)

#### start.py
- `/start` — приветствие и главное меню
- `/help` — подробная справка
- `/menu` — открытие меню
- `/stats` — быстрая статистика

#### chats.py
- `/add_chat` — добавление чата через пересылку сообщения
- `/list_chats` — список всех чатов
- Включение/выключение мониторинга
- Управление белым/чёрным списками
- Удаление чатов

#### accounts.py
- `/add_account` — добавление аккаунта (через диалог)
- `/list_accounts` — список аккаунтов
- Изменение стиля общения
- Активация/деактивация
- Удаление аккаунтов

## Команды бота

### Основные команды
- `/start` — запуск бота и главное меню
- `/help` — подробная справка по использованию
- `/menu` — открыть главное меню
- `/stats` — показать статистику
- `/cancel` — отменить текущее действие

### Команды управления чатами
- `/add_chat` — добавить чат в мониторинг
- `/list_chats` — список всех чатов

### Команды управления аккаунтами
- `/add_account` — добавить новый аккаунт
- `/list_accounts` — список всех аккаунтов

## Запуск

## Конфигурация

Бот использует следующие переменные окружения из `.env`:

```env
# Admin Bot
ADMIN_BOT_TOKEN=<токен от @BotFather>
OPERATOR_USER_ID=<ваш Telegram user_id>

# База данных
DATABASE_URL=postgresql+asyncpg://user:pass@host:port/db

# AI
OPENROUTER_API_KEY=<ключ OpenRouter>
AI_MODEL_PRIMARY=meta-llama/llama-3.3-70b-instruct:free

# Логирование
LOG_LEVEL=INFO
LOG_FILE=logs/leadhunter.log
```

## Логирование

Логи записываются:
- В консоль (stdout)
- В файл `logs/leadhunter.log`

Уровень логирования настраивается через `LOG_LEVEL` в `.env`.

## Безопасность

1. **Авторизация**: Бот доступен только оператору (проверка по `OPERATOR_USER_ID`)
2. **Валидация**: Все входные данные валидируются перед сохранением в БД
3. **Логирование**: Все действия логируются для аудита
4. **Обработка ошибок**: Все критичные операции обёрнуты в try-except

## Стили общения

Система поддерживает 3 стиля общения для аккаунтов:

- 🎩 **Вежливый/Деловой** — официальный тон, уважительное обращение
- 😊 **Дружеский** — неформальное дружелюбное общение  
- 💪 **Агрессивный** — напористый стиль, давление на срочность

Стиль можно изменить в любой момент через меню аккаунта.

## User Flow

### Добавление чата

1. Пользователь отправляет `/add_chat`
2. Пересылает сообщение из нужного чата
3. Бот извлекает информацию и сохраняет чат в БД
4. Чат автоматически включается в мониторинг

### Добавление аккаунта

1. Пользователь отправляет `/add_account`
2. Вводит название аккаунта (label)
3. Вводит номер телефона
4. Выбирает стиль общения из inline-клавиатуры
5. Аккаунт сохраняется в БД (но требует авторизации в Lead Listener)

### Работа с лидом

1. Получение уведомления о новом лиде
2. Просмотр лид-карточки с анализом ИИ
3. Выбор аккаунта и стиля
4. Просмотр/редактирование текста ответа
5. Отправка сообщения одной кнопкой

## Troubleshooting

### Бот не запускается

1. Проверьте правильность `ADMIN_BOT_TOKEN` в `.env`
2. Убедитесь, что БД доступна
3. Проверьте логи в `logs/leadhunter.log`

### Бот не отвечает

1. Убедитесь, что вы отправляете команды от правильного user_id
2. Проверьте `OPERATOR_USER_ID` в `.env`
3. Проверьте, что бот запущен

### Ошибки при добавлении чата

1. Убедитесь, что пересылаете сообщение из группы/канала
2. Проверьте, что чат не добавлен ранее
3. Для каналов используйте публичную ссылку

### Ошибки при добавлении аккаунта

1. Проверьте формат номера телефона (+7XXXXXXXXXX)
2. Убедитесь, что название (label) уникально
3. После добавления нужна авторизация в Lead Listener
