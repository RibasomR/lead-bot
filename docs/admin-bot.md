# Документация Admin Bot

## Обзор

Admin Bot — это центральный интерфейс управления системой LeadHunter, реализованный на aiogram 3.x. Предоставляет удобный Telegram-интерфейс для всех операций системы.

## Реализованный функционал

### ✅ Фаза 3.1: Базовая структура

**Файлы:**
- `admin_bot/main.py` — точка входа и инициализация
- `admin_bot/filters.py` — фильтры безопасности

**Возможности:**
- Инициализация Bot и Dispatcher с aiogram 3.x
- HTML parse mode по умолчанию
- OperatorFilter для доступа только авторизованному оператору
- Команды `/start` и `/help`
- Главное меню с разделами
- Уведомления о запуске/остановке бота

### ✅ Фаза 3.2: Управление чатами

**Файлы:**
- `admin_bot/handlers/chats.py` — обработчики чатов
- `admin_bot/states.py` — FSM состояния

**Команды:**
- `/add_chat` — добавление чата через пересылку сообщения
- `/list_chats` — список всех чатов с фильтрами

**Функции:**
- Включение/выключение мониторинга чата
- Добавление в белый список (приоритетные чаты)
- Добавление в чёрный список (игнорируемые чаты)
- Удаление чата с подтверждением
- Просмотр карточки чата с деталями

**Процесс добавления чата:**
1. Команда `/add_chat` или кнопка "Добавить чат"
2. Выбор способа:
   
   **Способ 1: По Chat ID или username** (рекомендуется для приватных групп)
   - Отправить Chat ID (например: `-1001234567890`)
   - Или username (например: `@pythonru`)
   - Система автоматически получит информацию о чате через Telethon
   
   **Способ 2: Пересылка сообщения** (только для публичных каналов)
   - Переслать любое сообщение из публичного канала
   - ⚠️ Для приватных групп не работает (Telegram скрывает информацию)

3. Автоматическое извлечение:
   - Telegram Chat ID
   - Название чата
   - Тип (group/supergroup/channel)
   - Username (если есть)
4. Сохранение в БД с приоритетом 1
5. Автоматическое включение мониторинга

**Как узнать Chat ID:**
- Добавьте бота @getmyid_bot в группу
- Напишите команду `/id`
- Скопируйте Chat ID (начинается с минуса)

### ✅ Фаза 3.3: Управление аккаунтами

**Файлы:**
- `admin_bot/handlers/accounts.py` — обработчики аккаунтов

**Команды:**
- `/add_account` — пошаговое добавление аккаунта
- `/list_accounts` — список всех аккаунтов

**Функции:**
- Привязка стиля общения (вежливый/дружеский/агрессивный)
- Активация/деактивация аккаунта
- Изменение стиля в любой момент
- Удаление аккаунта с подтверждением
- Просмотр карточки аккаунта

**Процесс добавления аккаунта:**
1. Команда `/add_account`
2. Ввод названия (label) — уникальное имя
3. Ввод номера телефона (+7XXXXXXXXXX)
4. Выбор стиля из inline-клавиатуры
5. Сохранение в БД (статус: неактивен)
6. Требуется авторизация в Lead Listener

**Стили общения:**
- 🎩 **Вежливый/Деловой** (`polite`)
  - Официальный тон
  - Уважительное обращение
  - Подходит для корпоративных заказчиков
  
- 😊 **Дружеский** (`friendly`)
  - Неформальное общение
  - Дружелюбный тон
  - Подходит для стартапов и личных проектов
  
- 💪 **Агрессивный** (`aggressive`)
  - Напористый стиль
  - Давление на срочность
  - Подходит для конкурентных ситуаций

### ✅ Фаза 3.4: Отображение лид-карточек

**Файлы:**
- `admin_bot/lead_card.py` — генерация карточек
- `admin_bot/keyboards.py` — клавиатуры

**Функции формирования карточек:**

#### `format_lead_card(lead, chat, ai_data)`
Создаёт красиво отформатированную HTML-карточку лида:

**Содержимое:**
- 🎯 Заголовок (из AI или "Новый лид")
- 💬 Информация о чате (название, тип, username)
- 👤 Информация об авторе сообщения
- 📝 Текст сообщения (до 800 символов)
- 🔗 Ссылка на оригинальное сообщение
- 🌐 Язык сообщения
- 🔧 Извлечённый стек технологий
- 📅 Дата и время создания лида
- 🏷️ ID лида

**Анализ ИИ (если доступен):**
- 📊 Оценка качества (1-5 звёзд)
- 🎨 Рекомендуемый тон общения
- 💰 Вилка цен (мин-макс)

#### `get_lead_card_keyboard(lead_id, accounts, has_ai_data)`
Создаёт inline-клавиатуру с действиями:

**Кнопки выбора аккаунта:**
- До 3 кнопок с активными аккаунтами
- Формат: «🎩 Основной», «😊 Резервный», «💪 Жёсткий»
- Автоматический выбор emoji по стилю аккаунта

**Кнопки ИИ:**
- 🤖 Запросить анализ ИИ (если нет данных)
- 🔄 Обновить анализ ИИ (если данные устарели)
- 💬 Варианты ответов (показ предложенных текстов)

**Кнопки действий:**
- ✏️ Свой текст — ввод произвольного текста
- ⚡ Быстрая отправка — отправка с настройками по умолчанию

**Навигация:**
- ⬅️ Предыдущий лид
- ➡️ Следующий лид
- 🚫 Игнорировать лид
- 🔙 К списку лидов

#### `get_reply_variants_keyboard(lead_id, num_variants)`
Клавиатура выбора варианта ответа от ИИ:
- ✉️ Вариант 1, 2, 3...
- 🔙 Назад к карточке

#### `get_send_confirmation_keyboard(lead_id, account_id, variant_index)`
Подтверждение отправки:
- ✅ Отправить
- ✏️ Редактировать текст
- ❌ Отменить

#### `format_leads_list(leads, page, total_pages)`
Список лидов с иконками статуса:
- 🆕 Новые (status: new)
- 👁 Просмотренные (status: viewed)
- ✅ Отвеченные (status: replied)
- 🚫 Игнорированные (status: ignored)

## Безопасность

### OperatorFilter

Все handlers защищены фильтром `OperatorFilter`:

```python
@router.message(Command("start"), OperatorFilter())
async def cmd_start(message: Message):
    ...
```

**Логика:**
1. Проверяет `message.from_user.id` или `callback.from_user.id`
2. Сравнивает с `settings.operator_user_id`
3. Пропускает только совпадающие ID

**Защита:**
- ❌ Посторонние не могут использовать бота
- ❌ Нет доступа к данным системы
- ✅ Все неавторизованные запросы отклоняются
- ✅ Логируется попытка доступа

## Логирование

### Конфигурация

```python
# Уровень: DEBUG, INFO, WARNING, ERROR, CRITICAL
LOG_LEVEL=INFO
LOG_FILE=logs/leadhunter.log
```

### Формат логов

```
2025-11-15 12:34:56 - admin_bot - INFO - main.py:45 - ✅ Bot запущен
2025-11-15 12:35:01 - admin_bot - INFO - start.py:23 - Команда /start от оператора
```

### Категории логов

- **INFO** — успешные операции, старт/стоп
- **WARNING** — некритичные проблемы
- **ERROR** — ошибки в обработчиках
- **CRITICAL** — критичные сбои системы

## Обработка ошибок

### Стратегия

1. **Валидация входных данных**
   - Проверка форматов
   - Проверка уникальности
   - Проверка длины строк

2. **Try-except блоки**
   - Все операции с БД обёрнуты
   - Логирование исключений
   - Информирование пользователя

3. **Graceful degradation**
   - Если AI недоступен — работа без него
   - Если БД недоступна — уведомление оператору
   - Автоматические retry при временных сбоях

### Примеры обработки

```python
try:
    async with get_session() as session:
        chat = await create_chat(session, ...)
        await session.commit()
    await message.answer("✅ Чат добавлен")
except Exception as e:
    logger.error(f"Ошибка добавления чата: {e}")
    await message.answer("❌ Ошибка при добавлении чата")
```

## Расширение

### Добавление нового раздела

1. Создать `handlers/new_section.py`:
```python
router = Router(name="new_section_router")

@router.message(Command("new_command"), OperatorFilter())
async def cmd_new(message: Message):
    ...
```

2. Подключить в `handlers/__init__.py`:
```python
from . import new_section

def setup_handlers() -> Router:
    router = Router()
    router.include_router(new_section.router)
    return router
```

3. Добавить клавиатуру в `keyboards.py`

4. Добавить состояния в `states.py` (если нужны)

### Добавление middleware

```python
# В main.py
from aiogram import BaseMiddleware

class MyMiddleware(BaseMiddleware):
    async def __call__(self, handler, event, data):
        # До обработчика
        result = await handler(event, data)
        # После обработчика
        return result

dispatcher.message.middleware(MyMiddleware())
```

## Известные ограничения

1. **FSM в памяти**
   - Данные теряются при перезапуске
   - Решение: Redis storage (в планах)

2. **Одиночный оператор**
   - Только один авторизованный пользователь
   - Решение: мультиоператор (не в MVP)

3. **Нет обработки лидов**
   - Только просмотр карточек (фаза 3.4)
   - Отправка сообщений — фаза 3.5

4. **Нет истории с фильтрацией**
   - Только базовый список — фаза 3.6
